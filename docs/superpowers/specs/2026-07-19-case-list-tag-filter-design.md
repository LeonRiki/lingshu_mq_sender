# 测试用例列表标签筛选设计

## 目标

缩小测试用例列表文本搜索范围，并新增可多选的标签筛选。

## 筛选规则

- 文本搜索仅匹配测试场景名称和触发消息内容，不匹配业务场景、标签或请求次数。
- 业务场景继续使用单选精确筛选。
- 标签筛选支持多选；每条用例必须同时包含所有已选标签。
- 文本、业务场景和标签筛选使用 AND 组合。
- 任一筛选条件变化时，列表返回第一页。

## 界面

在测试用例列表工具栏中，业务场景筛选之后、文本搜索之前增加标签多选下拉。

- 选项来自当前启用的用户标签。
- 下拉支持清空已选标签。
- 宽度为 220px，与现有筛选控件并列展示。

## 实现边界

- 仅修改 `v1/web/app.js`、`v1/web/detail-ui.jsx` 和构建产物。
- 不修改版本号、README 或服务端接口。
- 使用现有 Ant Design `Select` 与列表工具栏事件机制。

## 验证

- 文本搜索不再命中业务场景和标签。
- 选中多个标签时，仅显示包含全部标签的用例。
- 标签筛选与业务场景、文本搜索可组合使用。
- 清空标签筛选后恢复不按标签过滤的结果。
